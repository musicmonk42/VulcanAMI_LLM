# Migration Notes
################################################################################

From v1.x to v2.0:
  1. Update OPA to 0.60.0+
  2. Review and update data.json with new fields
  3. Test new compliance features
  4. Enable audit logging
  5. Configure monitoring endpoints
  6. Update client integration (new decision format)
  7. Train users on new features
  8. Run migration validation tests

Breaking Changes:
  - Decision result format changed (now includes metadata)
  - New required fields in input (user.id, source.id)
  - Rate limiting enabled by default
  - Audit logging mandatory for PII data
  - Geographic restrictions enforced

Deprecated Features:
  - Simple allow/deny format (use decision.allow)
  - Direct DQS threshold comparison (use dqs_check_passed)
  - Single-tier rate limiting (now multi-tier)
